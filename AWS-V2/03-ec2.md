# EC2 - Elastic Compute Cloud

## 1. What is EC2?

Amazon Elastic Compute Cloud (EC2) is a **web service providing resizable compute capacity** — virtual machines in the cloud.

- **Elastic** — Scale up (add resources) or down (remove) instantly
- **On-demand** — Pay only for running instances
- **Self-service** — Launch, manage, terminate anytime
- **Secure** — Control network access via security groups + key pairs
- **Highly available** — Deploy across multiple AZs for fault tolerance

### Why EC2?

- No physical server procurement (months to weeks)
- Instant provisioning (minutes)
- Flexible resizing (vertical: change type; horizontal: add more instances)
- Pay-per-use (only for running hours)

## 2. Instance Types

EC2 offers dozens of instance types optimized for different use cases.

### 2.1 Instance Type Breakdown

**Format**: `family-type.size`

Example: `t3.large` → `t3` (family) + `.large` (size)

### 2.2 Instance Families

| Family | Performance | Use Case | Example |
|--------|-------------|----------|---------|
| **T3 (Burstable)** | General, variable | Dev/test, low traffic | t3.micro, t3.small |
| **M6i (General)** | Balanced CPU/RAM | Web servers, apps | m6i.large, m6i.xlarge |
| **C6i (Compute)** | High CPU | Batch processing, analytics | c6i.large |
| **R6i (Memory)** | High RAM | Databases, in-memory cache | r6i.large, r6i.2xlarge |
| **G4dn (GPU)** | GPU compute | ML, graphics, video encoding | g4dn.xlarge |
| **I3 (Storage)** | High I/O | NoSQL databases, data warehousing | i3.large |

### 2.3 Instance Sizes

```
nano < micro < small < medium < large < xlarge < 2xlarge < 4xlarge < ...
```

- **Nano/Micro**: 0.5–1 GB RAM, 1 vCPU (free tier eligible)
- **Small**: 2 GB RAM, 1 vCPU
- **Medium**: 4 GB RAM, 1–2 vCPU
- **Large**: 8 GB RAM, 2 vCPU
- **XLarge+**: Scale up to 128 GB RAM, 48 vCPU

### Common Interview Choices

- **Web server**: t3.small / t3.medium (burstable)
- **Production app**: m5.large / m5.xlarge (general purpose)
- **Database**: r5.large / r5.xlarge (memory optimized)
- **Batch job**: c5.large (compute optimized)
- **ML training**: g4dn.xlarge (GPU)

## 3. Launching an Instance

### 📟 Console — Launch an EC2 Instance

```
1. EC2 → Instances → Launch instance
2. Name: my-web-server (optional)
3. AMI (Amazon Machine Image): Ubuntu 22.04 LTS or Amazon Linux 2
4. Instance type: t3.micro (free tier eligible) or t3.small
5. Key pair: 
   - Create new → key-name → download .pem file (save safely!)
   - OR select existing
6. Network: Default VPC (or custom VPC)
7. Subnet: Any AZ (or specific for HA across AZs)
8. Auto-assign Public IP: Enable (to access from internet)
9. Security group:
   - Create or select
   - Add inbound rules:
     ☑ SSH (port 22) from your IP only
     ☑ HTTP (port 80) from 0.0.0.0/0 (internet)
     ☑ HTTPS (port 443) from 0.0.0.0/0 (internet)
10. Storage:
    - Root volume: gp3 or gp2, 20–100 GB (depends on OS + app)
    - Add additional EBS volumes if needed
11. Tags:
    - Name: my-web-server
    - Environment: dev / prod
12. Advanced:
    - User data: (optional) bash script to run on startup
13. → Launch instance
```

### 💻 CLI — Launch an Instance

```bash
# Basic launch
aws ec2 run-instances \
  --image-id ami-0c55b159cbfafe1f0 \
  --instance-type t3.micro \
  --key-name my-key-pair \
  --security-groups my-security-group \
  --region ap-south-1

# More detailed
aws ec2 run-instances \
  --image-id ami-0c55b159cbfafe1f0 \
  --instance-type t3.micro \
  --key-name my-key-pair \
  --security-group-ids sg-0123456789abcdef0 \
  --subnet-id subnet-0123456789abcdef0 \
  --tag-specifications 'ResourceType=instance,Tags=[{Key=Name,Value=my-server}]' \
  --user-data file://init.sh

# With user data script (init.sh)
#!/bin/bash
#!/bin/bash
sudo yum update -y
sudo yum install -y httpd
sudo systemctl start httpd
echo '<h1>Hello from EC2!</h1>' | sudo tee /var/www/html/index.html
```

## 4. Security Groups

A **security group is a virtual firewall** controlling inbound and outbound traffic.

### Key Concepts

- **Stateful**: If you allow inbound traffic, outbound is automatically allowed (return traffic)
- **Multiple rules**: Can have many rules per security group
- **Multiple SGs**: An instance can belong to multiple security groups
- **Inbound default**: All traffic is blocked by default (deny-by-default)
- **Outbound default**: All traffic is allowed by default

### Rule Structure

| Direction | Protocol | Port | Source/Destination | Reason |
|-----------|----------|------|-------------------|--------|
| **Inbound** | TCP | 22 | YOUR_IP/32 | SSH from you |
| **Inbound** | TCP | 80 | 0.0.0.0/0 | HTTP from internet |
| **Inbound** | TCP | 443 | 0.0.0.0/0 | HTTPS from internet |
| **Inbound** | TCP | 3306 | 10.0.0.0/8 | MySQL from internal |
| **Outbound** | TCP | 443 | 0.0.0.0/0 | HTTPS out (default) |

### 📟 Console — Add Inbound Rule

```
1. EC2 → Security Groups → select group
2. Inbound rules → Edit inbound rules
3. Add rule:
   - Type: SSH / HTTP / HTTPS / MySQL / Custom TCP
   - Port: auto-populated if type selected
   - Source: 
     - My IP (your current IP only)
     - Anywhere (0.0.0.0/0 — not recommended for SSH!)
     - Another SG (sg-xyz) for internal communication
4. → Save rules
```

### 💻 CLI — Manage Security Groups

```bash
# Create security group
aws ec2 create-security-group \
  --group-name my-sg \
  --description "Web server security group" \
  --vpc-id vpc-0123456789abcdef0

# Add inbound rule (SSH)
aws ec2 authorize-security-group-ingress \
  --group-id sg-0123456789abcdef0 \
  --protocol tcp \
  --port 22 \
  --cidr YOUR_IP/32

# Add inbound rule (HTTP)
aws ec2 authorize-security-group-ingress \
  --group-id sg-0123456789abcdef0 \
  --protocol tcp \
  --port 80 \
  --cidr 0.0.0.0/0

# Revoke rule
aws ec2 revoke-security-group-ingress \
  --group-id sg-0123456789abcdef0 \
  --protocol tcp \
  --port 22 \
  --cidr YOUR_IP/32
```

## 5. Key Pairs

A **key pair** is an SSH credential for secure login.

- **Public key**: Stored on EC2 instance (`~/.ssh/authorized_keys`)
- **Private key**: Downloaded as `.pem` file, kept on your computer
- **Access**: `ssh -i my-key.pem ec2-user@public-ip` (EC2 or ALB backed instances typically use `ec2-user`, Ubuntu uses `ubuntu`)

### 📟 Console — Create Key Pair

```
1. EC2 → Key Pairs → Create key pair
2. Name: my-key-pair
3. Type: RSA (or ED25519 for newer systems)
4. Format: .pem (for SSH) or .ppk (for PuTTY on Windows)
5. → Create → browser downloads .pem file
6. Secure: chmod 400 my-key-pair.pem
```

### 💻 CLI — Access Instance

```bash
# SSH into instance
ssh -i my-key-pair.pem ec2-user@54.123.45.67

# Or use the instance's private IP if accessing from same VPC
ssh -i my-key-pair.pem ec2-user@10.0.1.100

# Copy file to instance (scp)
scp -i my-key-pair.pem ./myfile.txt ec2-user@54.123.45.67:/tmp/

# Copy file from instance
scp -i my-key-pair.pem ec2-user@54.123.45.67:/tmp/file.txt ./
```

## 6. Elastic IP

An **Elastic IP (EIP)** is a static public IP address that doesn't change when you stop/start an instance.

- Public IP without EIP: changes if you stop the instance
- EIP: persistent, survives stop/start → good for production

### 📟 Console — Allocate & Associate EIP

```
1. EC2 → Elastic IPs → Allocate address
2. Scope: VPC
3. → Allocate
4. Select the new EIP → Associate address
5. Instance: select your instance
6. Private IP: auto-selected
7. → Associate
```

### 💻 CLI

```bash
# Allocate EIP
aws ec2 allocate-address --domain vpc

# Associate with instance
aws ec2 associate-address \
  --instance-id i-0123456789abcdef0 \
  --allocation-id eipalloc-0123456789abcdef0

# Disassociate
aws ec2 disassociate-address \
  --association-id eipassoc-0123456789abcdef0

# Release EIP (when no longer needed)
aws ec2 release-address --allocation-id eipalloc-0123456789abcdef0
```

## 7. EC2 Pricing Models

### 7.1 On-Demand

- Pay per hour (or per second for some new instances)
- No commitment, can terminate anytime
- Most expensive option

### 7.2 Reserved Instances (RI)

- Commit 1 year (33% discount) or 3 years (72% discount)
- Fixed instance type, family, region
- Good for baseline/steady-state workloads

### 7.3 Savings Plans

- Hourly commitment (1 or 3 years)
- Works across any instance type/family in region
- More flexible than RI, apply to EC2, Lambda, Fargate

### 7.4 Spot Instances

- Up to 90% discount vs on-demand
- AWS can terminate with 2-minute warning
- Good for: batch jobs, non-critical workloads, flexible timing

### 7.5 Dedicated Hosts

- Physical server dedicated to you
- For licensing, compliance, or performance isolation
- Most expensive

### 📊 Example Pricing (us-east-1, t3.large)

| Model | Price/hour | Cost/month (730 hrs) | Savings |
|-------|-----------|----------------------|---------|
| On-Demand | $0.1035 | $75.56 | — |
| Reserved (1y) | $0.070 | $51.10 | 32% ↓ |
| Reserved (3y) | $0.053 | $38.69 | 49% ↓ |
| Spot | $0.031 | $22.63 | 70% ↓ |

### Interview Tip

**Q: When should you use each model?**
- **On-Demand**: Dev/test, unpredictable workloads
- **Reserved**: Production baseline, steady-state
- **Savings Plans**: Multi-service flexibility (EC2 + Lambda + Fargate)
- **Spot**: Batch jobs, ML training, non-critical

## 8. Elastic Block Store (EBS)

EBS provides **persistent block storage** for EC2 instances.

### 8.1 Volume Types

| Type | Performance | Use Case | Cost |
|------|-------------|----------|------|
| **gp3** | General purpose, 3 IOPS/GB | Web servers, databases | Moderate |
| **gp2** | General purpose, burst | Legacy workloads | Low |
| **io1/io2** | Provisioned IOPS, high performance | High-traffic databases | High |
| **st1** | Throughput-optimized | Streaming workloads | Moderate |
| **sc1** | Cold multi-data | Infrequent access | Low |

### 8.2 Root vs Data Volumes

- **Root volume**: OS and system files. Terminates with instance by default (can be changed).
- **Data volumes**: Additional storage. Persists after instance termination.

### 📟 Console — Attach EBS Volume

```
1. EC2 → Volumes → Create volume
2. Type: gp3 or gp2
3. Size: 100 GB (example)
4. AZ: Must match instance AZ
5. → Create
6. Volumes → select volume → Attach
7. Instance: select your instance
8. Device name: /dev/sdf (auto-suggested)
9. → Attach
10. SSH into instance, partition, format, mount:
    sudo lsblk
    sudo mkfs -t ext4 /dev/xvdf
    sudo mkdir /data
    sudo mount /dev/xvdf /data
```

### 💻 CLI

```bash
# Create EBS volume
aws ec2 create-volume \
  --size 100 \
  --volume-type gp3 \
  --availability-zone ap-south-1a

# Attach to instance
aws ec2 attach-volume \
  --volume-id vol-0123456789abcdef0 \
  --instance-id i-0123456789abcdef0 \
  --device /dev/sdf

# Create snapshot (backup)
aws ec2 create-snapshot \
  --volume-id vol-0123456789abcdef0 \
  --description "Daily backup"

# Create volume from snapshot
aws ec2 create-volume \
  --snapshot-id snap-0123456789abcdef0 \
  --availability-zone ap-south-1a \
  --volume-type gp3
```

## 9. AMI (Amazon Machine Image)

An **AMI is a template** containing the OS, software, and configurations.

- **Public AMIs**: AWS-provided or community-shared
- **Custom AMIs**: Create from an EC2 instance
- **Marketplace**: Third-party AMIs (sometimes commercial)

### 📟 Console — Create Custom AMI

```
1. Launch an EC2 instance and customize it
2. EC2 → Instances → right-click instance
3. Image and templates → Create image
4. Image name: my-custom-ami
5. Description: Web server with Node.js pre-installed
6. → Create image
7. AMIs → view status (pending → available)
```

### 💻 CLI

```bash
# Create AMI from instance
aws ec2 create-image \
  --instance-id i-0123456789abcdef0 \
  --name my-custom-ami \
  --description "Web server with Node.js"

# Use AMI to launch instance
aws ec2 run-instances \
  --image-id ami-0customami12345678 \
  --instance-type t3.micro
```

## 10. Instance Lifecycle

### States

```
pending → running → stopping → stopped
            ↓           ↓
          terminated (cannot restart)
            ↓
          rebooting (same instance, same IPs)
```

- **Pending**: Instance is being launched
- **Running**: Ready for use
- **Stopping**: Graceful shutdown in progress
- **Stopped**: Can restart (EBS volumes retained)
- **Terminated**: Deleted (cannot restart, but EBS persists if not set to delete on terminate)
- **Rebooting**: Like a hard restart

### Costs

- **Running**: Charged per hour (or second)
- **Stopped**: Only EBS storage charged, not compute
- **Terminated**: No charges (but data in S3/EBS still charged if retained)

## 11. Interview Q&A

**Q: What is the difference between a public and private IP?**
A public IP is routable from the internet. A private IP (10.x.x.x, 172.16.x.x, 192.168.x.x) is only accessible within the VPC or via VPN. Elastic IPs are static public IPs.

**Q: What happens if you stop an instance?**
The instance is paused (not running). You're not charged for compute. EBS volumes persist. Public IP may change (unless using Elastic IP). You can restart it later.

**Q: What is the difference between stopping and terminating?**
Stopping can be reversed; terminating deletes the instance. You can restart a stopped instance, but not a terminated one.

**Q: Why would you use a security group instead of a firewall?**
Security groups are AWS's network firewall. They're instance-level and easier than network firewalls (NACLs). They're stateful (return traffic auto-allowed).

**Q: How do you make an instance highly available?**
Launch 2+ instances across different AZs, put them behind an load balancer (ALB/NLB), and use Auto Scaling to replace failed instances.

**Q: What is the recommended way to bootstrap an instance?**
Use a user data script (startup script) to install software and configure the instance. Store sensitive data in Systems Manager Parameter Store or Secrets Manager, not in user data.

**Q: Why use an AMI?**
Faster deployment. Instead of installing software every time, build a custom AMI and launch instances from it (minutes vs hours).

**Q: Can you change instance type after launch?**
Yes, but the instance must be stopped. Stopping is quick. Choose a type in the same family or compatible type.

**Q: How do you secure SSH access?**
Use security groups (allow SSH only from your IP). Use strong key pairs. Rotate keys regularly. Disable password auth, use only key-based.

**Q: What is Spot Interruption?**
Spot instances can be terminated by AWS with 2-minute notice if spot price rises. Design for graceful shutdown (finish current job, save state, update load balancer).
