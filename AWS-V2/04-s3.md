# S3 - Object Storage

## 1. What is S3?

Amazon Simple Storage Service (S3) is an **object storage** service — scalable, highly durable, and accessible over HTTP/HTTPS.

- **Not a file system.** Objects live in a flat namespace (bucket). Key names with `/` simulate folders but there are no real directories.
- **Not block storage.** You can't mount S3 like a disk — that's EBS. You read/write entire objects.
- **Durability:** 99.999999999% (11 nines) — data is redundantly stored across multiple AZs.
- **Availability:** 99.99% for Standard class.
- **Max object size:** 5 TB. Single PUT limit: 5 GB (use Multipart Upload for > 100 MB).
- **No limit** on number of objects or total storage per bucket.

### 📟 Console — Create a Bucket

```
1. S3 → Buckets → Create bucket
2. Bucket name: my-unique-bucket-name  (globally unique, lowercase)
3. Region: ap-south-1  (choose closest to users)
4. Object Ownership: ACLs disabled (recommended)
5. Block Public Access: all 4 settings ON (default)
6. Versioning: Enable (optional at creation)
7. Encryption: SSE-S3 (default) or SSE-KMS
8. → Create bucket
```

### 💻 CLI

```bash
aws s3 mb s3://my-unique-bucket-name --region ap-south-1

# Upload a file
aws s3 cp file.txt s3://my-unique-bucket-name/

# List objects
aws s3 ls s3://my-unique-bucket-name/

# Delete object
aws s3 rm s3://my-unique-bucket-name/file.txt
```

## 2. Core Concepts

### 2.1 Buckets

- Logical container for objects.
- **Globally unique name** across all AWS accounts worldwide.
- Created in a specific AWS Region — data stays there unless replication is configured.
- Name rules: 3–63 lowercase characters, no uppercase, no underscores, no IPs, must start/end with letter or number.

### 2.2 Objects

- The actual data — a file plus its metadata.
- Identified by a **Key** (full path string), e.g. `images/profile/user123.jpg`.
- Metadata: system (Content-Type, Last-Modified) + user-defined (key-value pairs, up to 2 KB).
- A **Version ID** is assigned if versioning is enabled.

### 2.3 Keys

- Unique identifier for an object within a bucket.
- No real folders — `photos/2024/dog.jpg` is one flat key. Console simulates folders using `/` delimiter.

### 2.4 Regions

- Each bucket lives in one region. Choose the region closest to your users.
- Cross-region data transfer incurs cost.

### 2.5 ARN Format

```
arn:aws:s3:::my-bucket
arn:aws:s3:::my-bucket/path/to/object
```

### 📟 Console — Upload an Object

```
1. S3 → Buckets → my-bucket → Upload
2. Add files / Add folder
3. (Optional) Set storage class per object here
4. (Optional) Set server-side encryption override
5. → Upload
```

### 💻 CLI

```bash
# Upload with specific storage class
aws s3 cp report.pdf s3://my-bucket/reports/ --storage-class STANDARD_IA

# Download
aws s3 cp s3://my-bucket/reports/report.pdf ./local-copy.pdf

# Sync local folder to S3
aws s3 sync ./dist s3://my-bucket/frontend/
```

## 3. Storage Classes

Choose based on **access frequency**, **retrieval time**, and **cost tolerance**.

| Storage Class | Use Case | Availability | Min Storage Duration | Retrieval Fee |
|---|---|---|---|---|
| **S3 Standard** | Frequently accessed data | 99.99% | None | None |
| **S3 Intelligent-Tiering** | Unknown/changing patterns | 99.9% | None | None (monitoring fee/object) |
| **S3 Standard-IA** | Infrequent, rapid access needed | 99.9% | 30 days | Per GB |
| **S3 One Zone-IA** | Infrequent, non-critical (single AZ) | 99.5% | 30 days | Per GB |
| **S3 Glacier Instant Retrieval** | Archive, millisecond retrieval | 99.9% | 90 days | Per GB |
| **S3 Glacier Flexible Retrieval** | Archive, 1 min–12 hrs | 99.99% | 90 days | Per GB |
| **S3 Glacier Deep Archive** | Long-term archive, 12–48 hrs | 99.99% | 180 days | Per GB |
| **S3 Express One Zone** | Ultra-low latency (single AZ) | 99.95% | None | None |

**Key interview points:**
- Standard-IA charges a retrieval fee — bad choice for frequently accessed data.
- One Zone-IA loses data if the AZ goes down — never use for primary critical data.
- Intelligent-Tiering moves objects between tiers automatically (Frequent → Infrequent → Archive Instant). No retrieval fee, but a small per-object monitoring charge applies.
- Glacier Flexible Retrieval has 3 sub-tiers: Expedited (1–5 min), Standard (3–5 hrs), Bulk (5–12 hrs).

### 📟 Console — Change Storage Class of an Existing Object

```
1. S3 → Buckets → my-bucket → click object
2. Properties tab → Storage class → Edit
3. Select new class (e.g. Standard-IA)
4. → Save changes
```

### 💻 CLI

```bash
# Change storage class by copying object over itself
aws s3 cp s3://my-bucket/archive.zip s3://my-bucket/archive.zip \
  --storage-class GLACIER
```

## 4. Bucket Configuration

### 4.1 Block Public Access

S3 has **4 Block Public Access settings** (account-level and bucket-level):

1. **BlockPublicAcls** — Block new public ACLs being added.
2. **IgnorePublicAcls** — Ignore any existing public ACLs.
3. **BlockPublicPolicy** — Block new public bucket policies.
4. **RestrictPublicBuckets** — Restrict cross-account access to public buckets.

By default, all 4 are ON for new buckets. For static websites, you disable some.

Most critical topic for interviews.

Attached to **users, roles, or groups**. Controls what AWS principals can do with S3.

### 5.3 ACLs (Access Control Lists)

Legacy mechanism. Object-level and bucket-level. AWS now recommends **disabling ACLs** — set Object Ownership to "Bucket Owner Enforced".

### 5.4 Access Points

Named network endpoints with their own ARN, policy, and optional VPC restriction. Simplifies access management when many apps share one bucket.

### 📟 Console — Create an Access Point

```
1. S3 → Buckets → my-bucket → Access Points tab
2. → Create access point
3. Name: my-app-access-point
4. Network: Internet or VPC (choose VPC for private access)
5. Set access point policy (like a bucket policy)
6. → Create access point
```

### 5.5 S3 Object Ownership

- **Bucket Owner Enforced (recommended):** ACLs disabled, bucket owner owns all objects automatically.
- **Bucket Owner Preferred:** Objects uploaded with `bucket-owner-full-control` ACL go to bucket owner.
- **Object Writer:** Uploader owns the object (old default).

### 📟 Console — Set Object Ownership

```
1. S3 → Buckets → my-bucket → Permissions tab
2. Object Ownership → Edit
3. Select: Bucket owner enforced (recommended)
4. → Save changes
```

### 5.6 Policy Evaluation Order

1. **Explicit Deny** anywhere → always denied (overrides everything).
2. **Allow** in bucket policy OR IAM policy → allowed (same-account).
3. **Cross-account:** Both IAM policy AND bucket policy must allow.

### 5.7 VPC Endpoint for S3

Allows EC2 instances in a VPC to access S3 without going through the internet.

| Type | Cost | Works from on-premises? | Supported services |
|---|---|---|---|
| **Gateway Endpoint** | Free | No | S3, DynamoDB only |
| **Interface Endpoint (PrivateLink)** | Paid | Yes (via Direct Connect/VPN) | All AWS services |

### 📟 Console — Create Gateway Endpoint


## 6. Versioning

- **Disabled** (default): overwrite/delete is permanent.
- **Enabled:** every PUT creates a new version with a unique Version ID. DELETE only adds a **Delete Marker**.
- **Suspended:** stops new versions; existing versions are kept.

### Key Points

- To permanently delete: explicitly delete a specific Version ID.
- To restore a deleted object: delete its Delete Marker.
- **MFA Delete:** requires MFA to change versioning state or permanently delete a version. Extra security layer.
- Versioning cannot be fully disabled once enabled — only suspended.
- All versions consume storage and add to your bill.

### 📟 Console — Enable Versioning

```
1. S3 → Buckets → my-bucket → Properties tab
2. Bucket Versioning → Edit → Enable
3. → Save changes
```

### 📟 Console — View & Restore Versions

```
1. S3 → Buckets → my-bucket
2. Toggle "Show versions" (top right of object list)
3. To restore: delete the Delete Marker for the object
4. To permanently delete: select specific Version ID → Delete
```

### 💻 CLI

```bash
# List all versions of objects
aws s3api list-object-versions --bucket my-bucket

# Delete a specific version permanently
aws s3api delete-object \
  --bucket my-bucket \
  --key file.txt \
  --version-id "abc123versionId"
```

## 7. Lifecycle Policies

Automate moving or deleting objects based on age or version status — essential for cost management.

### 7.1 Transition Actions

Move objects between storage classes after N days.

```
Day 0   → S3 Standard             (new uploads)
Day 30  → S3 Standard-IA
Day 90  → S3 Glacier Flexible Retrieval
Day 365 → S3 Glacier Deep Archive
```

### 7.2 Expiration Actions

- Delete current object versions after N days.
- Delete expired Delete Markers (keeps the bucket clean).
- **Delete incomplete Multipart Uploads** after N days (very common cost-saving rule).

### 7.3 Rules

- Apply to entire bucket or filter by prefix/tag.
- Min storage durations still apply even with lifecycle rules (you get billed for them).

### 📟 Console — Create a Lifecycle Rule

```
1. S3 → Buckets → my-bucket → Management tab
2. Lifecycle rules → Create lifecycle rule
3. Rule name: archive-documents
4. Rule scope:
   - Apply to all objects  OR
   - Limit scope: prefix = documents/  (optional)
5. Lifecycle rule actions:
   ☑ Transition current versions between storage classes
      - Transition to Standard-IA:          after 30 days
      - Transition to Glacier Flexible:     after 90 days
      - Transition to Glacier Deep Archive: after 365 days
   ☑ Expire current versions of objects:   after 730 days (optional)
   ☑ Delete expired object delete markers  (recommended)
   ☑ Delete incomplete multipart uploads:  after 7 days  (recommended)
6. Review → Create rule
```

### 💻 CLI

```bash
# Apply lifecycle config from JSON file
aws s3api put-bucket-lifecycle-configuration \
  --bucket my-bucket \
  --lifecycle-configuration file://lifecycle.json
```

```json
{
  "Rules": [{
    "ID": "archive-documents",
    "Status": "Enabled",
    "Filter": { "Prefix": "documents/" },
    "Transitions": [
      { "Days": 30,  "StorageClass": "STANDARD_IA" },
      { "Days": 90,  "StorageClass": "GLACIER" },
      { "Days": 365, "StorageClass": "DEEP_ARCHIVE" }
    ],
    "Expiration": { "Days": 730 },
    "AbortIncompleteMultipartUpload": { "DaysAfterInitiation": 7 }
  }]
}
```

## 8. Replication

### 8.1 Cross-Region Replication (CRR)

Replicates objects to a bucket in **another region**. Use cases: disaster recovery, geo-compliance, lower read latency for global users.

### 8.2 Same-Region Replication (SRR)

Replicates objects to a bucket in the **same region**. Use cases: log aggregation, prod → test environment sync.

### 8.3 Requirements & Behaviour

- **Versioning must be enabled** on both source and destination buckets.
- Replication is **asynchronous** — not instant.
- Only replicates **new objects** after the rule is created. Existing objects need **S3 Batch Replication**.
- Delete Markers are **not replicated** by default (can be enabled per rule).
- Permanent version deletions are **never replicated** (safety by design).
- Requires an IAM role that S3 assumes to replicate on your behalf.

### 8.4 Replication Time Control (RTC)

Optional SLA-backed feature — 99.99% of objects replicate within 15 minutes. Comes with replication metrics and EventBridge notifications.

### 📟 Console — Set Up CRR

```
1. Both buckets: enable Versioning first (required)
2. S3 → Source bucket → Management tab
3. Replication rules → Create replication rule
4. Rule name: crr-to-backup
5. Status: Enabled
6. Scope: All objects  OR  filter by prefix/tag
7. Destination:
   - Choose bucket in another AWS Region
   - Bucket: arn:aws:s3:::my-backup-bucket
8. IAM role: Create new role (auto-generated) OR choose existing
9. Additional options:
   ☑ Replication Time Control (RTC) — for 15 min SLA
   ☑ Replicate delete markers (optional)
10. → Save
```

### 💻 CLI

```bash
aws s3api put-bucket-replication \
  --bucket source-bucket \
  --replication-configuration file://replication.json
```

## 9. S3 Performance & Optimisation

### 9.1 Baseline Throughput (per prefix)

- **3,500 PUT/COPY/POST/DELETE** requests/second
- **5,500 GET/HEAD** requests/second

Spread objects across multiple prefixes to multiply throughput linearly.

### 9.2 Key Naming (Post-2018)

S3 now auto-scales partitioning — sequential key names (date-based, incremental) are fine. Randomising prefixes is no longer required but can still help for extreme throughput.

### 9.3 Multipart Upload

For objects > 100 MB. Upload parts in parallel, re-upload only failed parts.

### 9.4 Transfer Acceleration

Upload via nearest CloudFront edge → AWS backbone → S3. Faster for distant clients.

### 9.5 Byte-Range Fetches

Download specific byte ranges of an object in parallel (like a download manager). Also used to read just the first few bytes of large files.

### 📟 Console — Enable Transfer Acceleration

```
1. S3 → Buckets → my-bucket → Properties tab
2. Transfer acceleration → Edit → Enable
3. → Save changes
4. New endpoint: my-bucket.s3-accelerate.amazonaws.com
```

### 💻 CLI — Byte-Range Fetch

```bash
# Download only bytes 0–1023 (first 1 KB) of a large file
aws s3api get-object \
  --bucket my-bucket \
  --key large-file.bin \
  --range "bytes=0-1023" \
  output-chunk.bin
```

## 12. S3 Select & Athena

### 12.1 S3 Select

Run SQL queries **on a single S3 object** (CSV, JSON, Parquet) without downloading the full file. Returns only matching rows.

- Up to 400% faster and 80% cheaper than full-file download + local filter.

```sql
SELECT * FROM S3Object WHERE cast(age AS INT) > 25
```

### 📟 Console — Run S3 Select

```
1. S3 → Buckets → my-bucket → click object (CSV/JSON/Parquet)
2. Object actions → Query with S3 Select
3. Input settings: format = CSV / JSON / Parquet, set delimiter / compression
4. Output settings: CSV or JSON
5. SQL expression: SELECT * FROM s3object WHERE age > 25
6. → Run SQL
7. Results shown inline / download option available
```

### 12.2 Amazon Athena

Serverless SQL engine for querying many S3 objects. Pay per TB scanned.

- Backed by Presto/Trino. Supports CSV, JSON, ORC, Avro, Parquet.
- Uses **Glue Data Catalog** as the schema/metadata store.
- Convert data to Parquet/ORC + compress = massive cost reduction.

### 📟 Console — Query S3 with Athena

```
1. Athena → Query editor
2. First time setup:
   Settings → Manage → set query result location: s3://my-athena-results/
3. Create table:
   Database → Create table → from S3 bucket data (wizard)
   OR write DDL directly:
```

```sql
CREATE EXTERNAL TABLE logs (
  ip        STRING,
  timestamp STRING,
  action    STRING
)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ','
LOCATION 's3://my-bucket/logs/';

-- Query
SELECT action, COUNT(*) AS cnt
FROM logs
GROUP BY action
ORDER BY cnt DESC;
```

## 13. Encryption

### 13.1 Encryption at Rest

| Method | Key managed by | Notes |
|---|---|---|
| **SSE-S3** | AWS (S3 manages key) | Default. AES-256. No extra cost. |
| **SSE-KMS** | AWS KMS | Audit trail via CloudTrail. Control key rotation. KMS API call costs apply. |
| **SSE-C** | Customer | You provide key per request. AWS never stores it. HTTPS required. |
| **Client-Side** | Customer | Encrypt before upload. AWS never sees plaintext. |

**S3 Bucket Keys:** Reduces KMS API calls by generating a short-lived data key at bucket level — cuts KMS cost by up to 99%.

### 📟 Console — Set Default Bucket Encryption

```
1. S3 → Buckets → my-bucket → Properties tab
2. Default encryption → Edit
3. Encryption type:
   - SSE-S3  (free, AWS-managed)
   - SSE-KMS → choose KMS key (AWS managed or customer managed)
     ☑ Bucket Key: Enable  ← always enable to reduce KMS cost
4. → Save changes
```

### 📟 Console — Upload with Encryption Override

```
1. S3 → Upload → Additional upload options (expand at bottom)
2. Server-side encryption → Override bucket settings
3. Choose: SSE-S3 or SSE-KMS + key ARN
4. → Upload
```

### 💻 CLI

```bash
# Upload with SSE-KMS
aws s3 cp secret.txt s3://my-bucket/ \
  --sse aws:kms \
  --sse-kms-key-id arn:aws:kms:region:account:key/key-id
```

### 13.2 Enforce HTTPS Only (Encryption in Transit)

Add this to your bucket policy:

```json
{
  "Effect": "Deny",
  "Principal": "*",
  "Action": "s3:*",
  "Resource": [
    "arn:aws:s3:::my-bucket",
    "arn:aws:s3:::my-bucket/*"
  ],
  "Condition": {
    "Bool": { "aws:SecureTransport": "false" }
  }
}
```

## 14. Consistency Model

As of **December 2020**, S3 provides **strong read-after-write consistency** for ALL operations.

- After a successful PUT → immediate GET returns the new object.
- After a successful DELETE → immediate GET returns 404.
- LIST is also strongly consistent.

**Before Dec 2020:** Eventual consistency for overwrite PUTs and DELETEs — a classic interview question. Know the history.

No special config needed — strong consistency is automatic for all buckets.

## 15. Multipart Upload

Splits large objects into parts and uploads them independently.

### Process

```
1. Initiate  → get Upload ID
2. Upload Parts (min 5 MB each, except last) → each returns an ETag
3. Complete  → S3 assembles the final object using all ETags
4. Abort     → cleans up incomplete parts (they still bill until aborted)
```

### Key Points

- Parts upload in parallel → faster than single-stream.
- Part size: 5 MB to 5 GB. Max 10,000 parts per upload.
- If a part fails, re-upload just that part.
- Use a lifecycle rule to auto-abort stuck uploads after N days.

### 💻 CLI

```bash
# High-level (handles multipart automatically for large files)
aws s3 cp large-file.zip s3://my-bucket/ \
  --multipart-threshold 100MB \
  --multipart-chunksize 64MB

# Low-level manual multipart
# Step 1: Initiate
UPLOAD_ID=$(aws s3api create-multipart-upload \
  --bucket my-bucket --key large-file.zip \
  --query 'UploadId' --output text)

# Step 2: Upload part 1
aws s3api upload-part \
  --bucket my-bucket --key large-file.zip \
  --part-number 1 --body part1.zip \
  --upload-id $UPLOAD_ID

# Step 3: Complete (pass ETags from each uploaded part)
aws s3api complete-multipart-upload \
  --bucket my-bucket --key large-file.zip \
  --upload-id $UPLOAD_ID \
  --multipart-upload file://parts.json

# Abort (cleanup)
aws s3api abort-multipart-upload \
  --bucket my-bucket --key large-file.zip \
  --upload-id $UPLOAD_ID
```

## 16. Pre-Signed URLs

Temporarily share access to a **private** S3 object without making the bucket public.

- Generated using the credentials of an IAM user or role.
- URL embeds signature, permissions, and expiry time.
- Anyone with the URL can access the object until it expires.
- **Max expiry:** 7 days (IAM user), 12 hours (IAM role via STS token).

### Use Cases

- Secure file downloads from private buckets.
- Let clients upload directly to S3 without routing through your server (pre-signed PUT URL).

### 📟 Console — Generate Pre-Signed URL

```
1. S3 → Buckets → my-bucket → click object
2. Object actions → Share with a presigned URL
3. Set expiry: 1 hour / 1 day / custom (max 7 days in console)
4. → Create presigned URL
5. Copy and share the URL
```

### 💻 CLI

```bash
# Pre-signed GET URL (valid for 1 hour)
aws s3 presign s3://my-bucket/private-report.pdf --expires-in 3600

# Pre-signed PUT URL (for direct upload)
aws s3 presign s3://my-bucket/uploads/user-photo.jpg \
  --expires-in 300 \
  --method PUT
```

## 17. S3 Object Lock & Glacier Vault Lock

### 17.1 Object Lock

Prevents objects from being deleted or overwritten for a set period. Implements **WORM** (Write Once Read Many). **Must be enabled at bucket creation.**

**Retention Modes:**

| Mode | Who can override? | Use Case |
|---|---|---|
| **Compliance** | No one — not even root | Strictest regulatory compliance (SEC, FINRA) |
| **Governance** | Users with `s3:BypassGovernanceRetention` | Internal policy, less strict |

**Legal Hold:** Indefinite WORM — no expiry date. Toggle on/off independently of retention period. Useful during litigation holds.

## 19. Pricing Model

### Cost Dimensions

- **Storage** — per GB/month (varies by storage class).
- **Requests** — per 1,000 requests (GET cheaper than PUT; varies by class).
- **Data Transfer Out** — to internet or other regions (in-region and to CloudFront is free).
- **Retrieval fee** — applies to Standard-IA, One Zone-IA, all Glacier classes.
- **Management & Analytics** — Inventory, Analytics, Object Tagging, Batch Operations.
- **Replication** — per GB replicated + destination storage cost.
- **KMS** — per KMS API call when using SSE-KMS (use Bucket Keys to reduce this).

### Cost Optimisation Checklist

```
☑ Lifecycle policies: Standard → IA → Glacier → Deep Archive
☑ Enable S3 Bucket Keys (cuts KMS cost up to 99%)
☑ Intelligent-Tiering for unpredictable access patterns
☑ Lifecycle rule: abort incomplete multipart uploads after 7 days
☑ S3 Select instead of downloading full objects
☑ S3 Storage Lens / Analytics to identify cold/stale data
☑ CloudFront in front of S3 (data transfer to CloudFront is free)
☑ Delete old non-current versions via lifecycle expiration rules
```

## 20. Interview Q&A — Quick Fire

**Q: What is the difference between S3, EBS, and EFS?**
S3 is object storage — unstructured data over HTTP, no mounting. EBS is block storage mounted to a single EC2 instance (like a hard disk). EFS is managed NFS mountable by many EC2 instances simultaneously.

**Q: Can two buckets have the same name?**
No. Bucket names are globally unique across all AWS accounts and all regions.

**Q: What happens when you delete an object in a versioned bucket?**
S3 adds a Delete Marker — the object isn't actually gone. To permanently delete, you must explicitly delete a specific Version ID.

**Q: How do you make a bucket private but share one object temporarily?**
Use a pre-signed URL. The bucket stays private; the URL embeds a valid signature for a limited time.

**Q: What is the difference between SSE-S3, SSE-KMS, and SSE-C?**
SSE-S3: AWS manages everything, no extra cost. SSE-KMS: AWS KMS manages the key — you control rotation and get audit logs via CloudTrail, but KMS API calls cost money. SSE-C: you provide the key per request; AWS never stores it; HTTPS required.

**Q: How do you serve a static website with HTTPS on a custom domain?**
Upload to S3, enable static website hosting, then front it with CloudFront (HTTPS + custom domain via ACM certificate). Use OAC so the S3 bucket stays private.

**Q: What is Intelligent-Tiering?**
A storage class that automatically moves objects between Frequent, Infrequent, and Archive Instant access tiers based on actual access patterns. No retrieval fee. Small per-object monitoring fee.

**Q: What is the minimum part size for Multipart Upload?**
5 MB per part (except the last part which can be any size). Recommended for files > 100 MB, required for files > 5 GB.

**Q: Can you replicate to multiple destination buckets?**
Yes. Configure multiple replication rules, each pointing to a different destination bucket in different regions.

**Q: What is S3 Object Lock used for?**
Implementing WORM storage for regulatory compliance. Objects cannot be deleted or overwritten during the retention period. Must be enabled at bucket creation.

**Q: What is strong consistency in S3 and when was it introduced?**
Since December 2020, S3 guarantees strong read-after-write consistency for all operations — PUT, DELETE, and LIST. Before that, overwrite PUTs and DELETEs had eventual consistency.

**Q: What is the maximum size of an S3 object?**
5 TB. Single PUT can upload up to 5 GB; anything larger requires multipart upload.

**Q: What is S3 Select?**
Runs SQL on a single S3 object (CSV, JSON, Parquet) and returns only matching rows. Saves bandwidth and cost vs downloading the full object.

**Q: How do you enforce HTTPS-only access?**
Add a bucket policy Deny on `aws:SecureTransport = false`. This blocks all HTTP requests.

**Q: Gateway Endpoint vs Interface Endpoint for S3?**
Gateway Endpoint is free, added as a route in the route table, only for S3 and DynamoDB, no on-premises support. Interface Endpoint uses PrivateLink (ENI), costs money, works for all services, accessible from on-premises via Direct Connect/VPN.

**Q: What is RTC (Replication Time Control)?**
An SLA-backed replication feature guaranteeing 99.99% of objects replicate within 15 minutes. Includes replication metrics and EventBridge notifications.

**Q: How does Transfer Acceleration work?**
Clients upload to the nearest CloudFront edge location. Data travels edge → S3 over AWS's fast private backbone, bypassing the slow public internet.

**Q: What are S3 Access Points?**
Named endpoints with their own ARN, policy, and optional VPC restriction — each attached to a bucket. Simplifies access management when multiple teams share one bucket.

**Q: What is S3 Batch Operations?**
A managed feature for running operations (copy, tag, invoke Lambda, restore from Glacier) on millions of S3 objects at once. Triggered by an S3 Inventory report or custom manifest.

**Q: What is S3 Bucket Key and why does it matter?**
A short-lived data encryption key generated at bucket level for SSE-KMS. Reduces KMS API calls by up to 99%, cutting KMS costs significantly for high-volume S3 usage.

**Q: How do you replicate existing objects (before the replication rule existed)?**
Use **S3 Batch Replication** — standard replication rules only capture new objects after the rule is created.
