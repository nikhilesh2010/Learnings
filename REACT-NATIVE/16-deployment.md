# 16: Deployment

## 🚀 Deployment Paths

```
Development
    │
    ├── Expo Go (instant preview)
    │
    ├── Development Build (full native features)
    │
    ├── Preview Build (internal testing)
    │
    └── Production Build
            ├── App Store (iOS)
            └── Play Store (Android)
```

---

## 🛠️ EAS (Expo Application Services)

EAS is the recommended build and distribution service for Expo apps.

```bash
# Install EAS CLI
npm install --global eas-cli

# Login
eas login

# Configure EAS for your project
eas build:configure
```

This generates `eas.json`:

```json
{
  "cli": {
    "version": ">= 5.0.0"
  },
  "build": {
    "development": {
      "developmentClient": true,
      "distribution": "internal"
    },
    "preview": {
      "distribution": "internal",
      "channel": "preview"
    },
    "production": {
      "channel": "production",
      "autoIncrement": true
    }
  },
  "submit": {
    "production": {}
  }
}
```

## 🏗️ Building

### Development Build

A full native build with dev tools — use instead of Expo Go for projects with custom native modules.

```bash
# Build for iOS device
eas build --profile development --platform ios

# Build for Android device
eas build --profile development --platform android

# Build for both
eas build --profile development --platform all
```

### Preview / Testing Build

Sharable builds for internal testers:

```bash
eas build --profile preview --platform android
# Generates APK/AAB that can be shared via link
```

### Production Build

```bash
# Build production
eas build --profile production --platform all

# Check build status
eas build:list
```

## 🍎 iOS App Store Submission

### Prerequisites
1. Apple Developer Account ($99/year): developer.apple.com
2. App created in App Store Connect
3. Bundle ID matching `app.json` → `ios.bundleIdentifier`

### First Submission

```bash
# Configure credentials (certificates + provisioning profiles)
eas credentials

# Build production IPA
eas build --profile production --platform ios

# Submit to App Store
eas submit --platform ios
```

### eas.json for iOS submit

```json
{
  "submit": {
    "production": {
      "ios": {
        "appleId": "your@email.com",
        "ascAppId": "1234567890",    // App Store Connect App ID
        "appleTeamId": "ABCDE12345"
      }
    }
  }
}
```

### App Store Review Checklist
- [ ] Privacy Policy URL in App Store Connect
- [ ] App icons at all required sizes (auto-generated by Expo)
- [ ] Screenshots for all device sizes
- [ ] Age rating configured
- [ ] No placeholder content
- [ ] All required permissions have usage descriptions in Info.plist
- [ ] TestFlight testing complete

## 🤖 Android Play Store Submission

### Prerequisites
1. Google Play Developer Account ($25 one-time)
2. App created in Google Play Console
3. Package name matching `app.json` → `android.package`

### Keystore (signing key)

```bash
# EAS manages this for you
eas credentials --platform android
# Select: "Set up a new keystore" → EAS stores it securely

# Or generate locally (store the file safely!)
keytool -genkey -v -keystore my-upload-key.keystore \
  -alias my-key-alias -keyalg RSA -keysize 2048 -validity 10000
```

> ⚠️ **Never lose your keystore file** — you cannot update an existing app without it.

### First Submission

```bash
# Build AAB (Android App Bundle — required for Play Store)
eas build --profile production --platform android

# Submit
eas submit --platform android
```

### eas.json for Android submit

```json
{
  "submit": {
    "production": {
      "android": {
        "serviceAccountKeyPath": "./play-store-service-account.json",
        "track": "production"
      }
    }
  }
}
```

**Tracks:** `internal` → `alpha` → `beta` → `production`

## 📡 OTA Updates (Over-the-Air)

Update your JS bundle without going through app stores.

```bash
# Publish OTA update on the "production" channel
eas update --channel production --message "Fix critical bug"

# Preview channel
eas update --channel preview --message "New feature test"
```

### Configure update channels in app.json

```json
{
  "expo": {
    "updates": {
      "url": "https://u.expo.dev/your-project-id",
      "enabled": true,
      "fallbackToCacheTimeout": 0,
      "checkAutomatically": "ON_LOAD"
    },
    "runtimeVersion": {
      "policy": "appVersion"
    }
  }
}
```

### Programmatic Updates

```javascript
import * as Updates from 'expo-updates';

async function checkForUpdates() {
  if (!Updates.isEmbeddedLaunch) {
    try {
      const update = await Updates.checkForUpdateAsync();
      if (update.isAvailable) {
        await Updates.fetchUpdateAsync();
        await Updates.reloadAsync();  // Reload app with new code
      }
    } catch (error) {
      console.error('Error checking for updates:', error);
    }
  }
}
```

> **OTA Limitations:** Only JS/assets update. Native code changes (new permissions, new native modules) require a full app store build.

## 🔖 App Versioning

```json
// app.json
{
  "expo": {
    "version": "1.2.0",          // User-visible version string
    "ios": {
      "buildNumber": "42"        // iOS internal build number (must increment)
    },
    "android": {
      "versionCode": 42          // Android version code (must increment)
    }
  }
}
```

**Auto-increment with EAS:**
```json
// eas.json
{
  "build": {
    "production": {
      "autoIncrement": true       // Automatically bumps buildNumber / versionCode
    }
  }
}
```

## 🔐 Environment Variables

```bash
# .env
EXPO_PUBLIC_API_URL=https://api.example.com
EXPO_PUBLIC_SENTRY_DSN=https://...

# Secret (not exposed to client)
MY_SECRET_KEY=super-secret
```

```javascript
// Access public env vars (prefixed with EXPO_PUBLIC_)
const apiUrl = process.env.EXPO_PUBLIC_API_URL;

// In EAS builds, set secrets in EAS dashboard or:
eas secret:create --scope project --name MY_SECRET_KEY --value "super-secret"
```

**app.config.js (dynamic config):**

```javascript
export default () => ({
  expo: {
    name: 'MyApp',
    extra: {
      apiUrl: process.env.EXPO_PUBLIC_API_URL,
      environment: process.env.APP_ENV ?? 'development',
    },
  },
});

// Access via:
import Constants from 'expo-constants';
const { apiUrl } = Constants.expoConfig.extra;
```
